# 분산에너지 보고서/자료 분석 및 포맷 가이드라인

## 1. 활성화 조건
- Always On (분산에너지 관련 보고서, 정책 자료, 논문 등 분석 문서 생성 및 수정 시 항시 적용)

## 2. 파일 생성 및 저장 규칙
- 보고서 분석 파일은 반드시 `reports/YYYY-MM-DD_보고서제목요약.md` 포맷으로 저장한다.
- 파일의 최상단에는 반드시 아래 규격의 YAML Frontmatter를 기입한다.
  ```yaml
  ---
  title: 보고서/자료 제목
  date: YYYY-MM-DD (발행일 혹은 분석일)
  author: 발행 기관 또는 저자 (예: 산업통상자원부, 에너지경제연구원 등)
  url: 원문 URL 또는 다운로드 링크
  tags: [태그1, 태그2]
  ---
  ```
- `tags` 항목에는 반드시 `.agents/rules/tags.md`에 정의된 표준 태그만 사용한다.

## 3. 보고서 분석 필수 구조 및 스타일
- **스타일 규정**: 문장은 명사형 종결 또는 간결체(예: '~함', '~임', '~됨')로 종결하며, 불필요한 서술형 어미(예: '~하였습니다', '~하고자 함')는 배제하고 최대한 요약식으로 작성한다.

# [분산에너지 보고서 분석] 보고서/자료 제목

- **분석일자:** YYYY-MM-DD
- **발행 기관/저자:** 발행 기관 또는 저자
- **원문 출처/링크:** [출처 표기](URL)

### 1. 자료의 배경 및 목적
- 이 보고서/자료가 발간된 핵심 배경 및 목적 요약 (간결체 작성)

### 2. 주요 핵심 내용 요약
- 핵심 쟁점, 제도 개편안, 기술적 솔루션 등 주요 내용 상세 기술 (요약식)

### 3. 이해관계자별 영향 분석
- **국가 관점**: 국가 에너지 정책 및 로드맵 정합성, 관련 법령 영향 등 분석
- **사업자 관점**: 시장 참여 사업자(발전사, VPP 공급자 등)에 대한 규제 완화 또는 리스크 분석
- **정책기관 관점**: KEPCO, KPX 등 규제 기관의 운영 방식 및 인프라 대응 요구사항 분석

### 4. 시사점 및 제언
- 향후 분산에너지 시장 활성화 및 사업 모델 개발에 주는 장기적 시사점 도출

## 4. 저장소 자동 등록 규칙
- 분석 결과 마크다운 파일이 생성되거나 수정되면, 에이전트는 즉시 다음 Git 명령을 수행하여 원격 저장소에 자동 등록해야 한다.
  1. 생성/수정된 파일을 스테이징한다: `git add reports/YYYY-MM-DD_보고서제목요약.md`
  2. 커밋을 생성한다: `git commit -m "docs: YYYY-MM-DD 보고서 분석 결과 추가"`
  3. 원격 저장소에 푸시한다: `git push`
