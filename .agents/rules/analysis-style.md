# 분산에너지 기사 분석 및 포맷 가이드라인

## 1. 활성화 조건
- Always On (분산에너지 관련 문서 생성 및 수정 시 항시 적용)

## 2. 파일 생성 및 저장 규칙
- 기사 파일은 반드시 `news/YYYY-MM-DD_기사제목요약.md` 포맷으로 저장한다.
- 파일의 최상단에는 반드시 아래 규격의 YAML Frontmatter를 기입한다.
  ```yaml
  ---
  title: 기사 제목
  date: YYYY-MM-DD
  source: 출처명 (예: 전자신문, 이투뉴스 등)
  url: 원문 URL
  tags: [태그1, 태그2]
  ---
  ```
- `tags` 항목에는 반드시 `.agents/rules/tags.md`에 정의된 표준 태그만 사용한다.

## 3. 기사 분석 필수 구조 및 스타일
- **스타일 규정**: 문장은 명사형 종결 또는 간결체(예: '~함', '~임', '~됨')로 종결하며, 불필요한 서술형 어미(예: '~하였습니다', '~하고자 함')는 배제하고 최대한 요약식으로 작성한다.

# [분산에너지 동향] 기사 제목

- **분석일자:** YYYY-MM-DD
- **원문 출처/링크:** [출처 표기](URL)

### 1. 기사 요약
- 핵심 내용 3줄 요약 (간결체 작성)

### 2. 주요 핵심 키워드
- 핵심 기술/정책 키워드 3~5개 (표준 태그와 매칭 권장)

### 3. 이해관계자별 관점 분석
- **국가 관점**: 정부의 분산에너지 활성화 로드맵 및 탄소중립 정책에 미치는 영향 분석
- **사업자 관점**: 민간 발전/VPP/ESS 사업자의 수익성, 규제 리스크, 비즈니스 모델 발굴 기회 등 분석
- **정책기관 관점**: KEPCO, KPX 등 전력계통 관리 기관의 인프라 구축 부담 및 제도 설계 관점 분석

### 4. 시장 및 정책적 시사점
- 단기/장기적인 비즈니스 모델 및 규제 변화 가능성에 대한 인사이트 도출

## 4. 저장소 자동 등록 규칙
- 분석 결과 마크다운 파일이 생성되거나 수정되면, 에이전트는 즉시 다음 Git 명령을 수행하여 원격 저장소에 자동 등록해야 한다.
  1. 생성/수정된 파일을 스테이징한다: `git add news/YYYY-MM-DD_기사제목요약.md`
  2. 커밋을 생성한다: `git commit -m "docs: YYYY-MM-DD 기사 분석 결과 추가"`
  3. 원격 저장소에 푸시한다: `git push`
