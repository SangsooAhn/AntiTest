# 분산에너지 기사 분석 및 포맷 가이드라인

## 1. 활성화 조건
- Always On (분산에너지 관련 문서 생성 및 수정 시 항시 적용)

## 2. 파일 생성 및 저장 규칙
- 기사 파일은 반드시 `news/YYYY-MM-DD_기사제목요약.md` 포맷으로 저장한다.
- 파일의 최상단에는 반드시 아래 규격의 YAML Frontmatter를 기입한다.
  ```yaml
  ---
  title: 기사 제목
  date: YYYY-MM-DD
  source: 출처명 (예: 전자신문, 이투뉴스 등)
  url: 원문 URL
  tags: [태그1, 태그2]
  ---
  ```
- `tags` 항목에는 반드시 `.agents/rules/tags.md`에 정의된 표준 태그만 사용한다.

## 3. 기사 분석 필수 구조 및 스타일
- **스타일 규정**: 문장은 명사형 종결 또는 간결체(예: '~함', '~임', '~됨')로 종결하며, 불필요한 서술형 어미는 배제하고 최대한 요약식으로 작성한다.
- **분석 핵심**: 기사 내 객관적인 '사실 관계(Facts)'와 이해관계자들이 도출하거나 파급되는 논리적 '명제(Propositions)'를 명확히 구분하여 작성한다.
- **명제 작성 규칙**: 명제는 주장하는 바가 제목에 명확히 드러나도록 **주요 키워드 3~5개 내외**를 사용하여 간결한 제목 형태(예: `[키워드 요약 제목]`)로 헤더를 구성하고 하단에 간결하게 서술한다.
- **관점 생략 허용**: 수집한 자료에서 특정 관점(예: 소비자 관점)으로 분석할 유의미한 내용이 없을 경우, 억지로 가공하지 않고 해당 관점 섹션은 완전히 생략한다.

# [기사 분석] 기사 제목

- **분석일자:** YYYY-MM-DD
- **원문 출처/링크:** [출처 표기](URL)

### 1. 사실 관계 (Facts)
- 기사에서 전달하는 객관적 사실, 일정, 법안 통과 등 입증 가능한 정보 요약 (각 사실마다 반드시 `[YYYY-MM-DD]` 형태로 발생 일자를 머리에 명시하고 3줄 이내 요약식 작성)

### 2. 주요 핵심 키워드
- 핵심 기술/정책 키워드 3~5개 (표준 태그와 매칭 권장)

### 3. 관점별 핵심 명제 (Propositions)
*(분석할 내용이 있는 관점만 선택하여 작성하며, 해당사항이 없는 관점은 섹션 자체를 생략함)*
- **국가 관점**: 
  - #### [키워드3~5개 요약 제목]
    - 세부 명제 내용 기술 (간결체)
- **사업자 관점**: 
  - #### [키워드3~5개 요약 제목]
    - 세부 명제 내용 기술 (간결체)
- **소비자 관점**: 
  - #### [키워드3~5개 요약 제목]
    - 세부 명제 내용 기술 (간결체)
- **정책기관 관점**: 
  - #### [키워드3~5개 요약 제목]
    - 세부 명제 내용 기술 (간결체)


## 4. 저장소 자동 등록 규칙
- 분석 결과 마크다운 파일이 생성되거나 수정되면, 에이전트는 즉시 다음 Git 명령을 수행하여 원격 저장소에 자동 등록해야 한다.
  1. 생성/수정된 파일을 스테이징한다: `git add news/YYYY-MM-DD_기사제목요약.md`
  2. 커밋을 생성한다: `git commit -m "docs: YYYY-MM-DD 기사 분석 결과 추가"`
  3. 원격 저장소에 푸시한다: `git push`
